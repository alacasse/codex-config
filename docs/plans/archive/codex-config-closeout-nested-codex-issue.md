## What Actually Broke

The outer runner already launched `closeout` as a fresh `codex exec` phase. Inside that closeout phase, the closeout agent tried to run additional nested Codex probes:

```bash
codex exec --ephemeral \
  --cd /home/alacasse/projects/graphify \
  --sandbox workspace-write \
  'Return {"ok":true} as plain JSON.'
```

That failed because `codex exec --ephemeral` still tried to initialize Codex state under the default Codex home:

```text
failed to open state DB at /home/alacasse/.codex/state_5.sqlite:
attempt to write a readonly database
Error: failed to initialize in-process app-server client:
Read-only file system (os error 30)
```

The closeout agent then created a temporary Codex home and tried again:

```bash
CODEX_HOME=/tmp/codex-runner-home codex exec \
  --cd /home/alacasse/projects/graphify \
  --sandbox workspace-write \
  'Return {"ok":true} as plain JSON.'
```

The temp home initialized, but the nested Codex call needed remote sampling and the closeout sandbox had restricted network. It retried against `chatgpt.com` and failed with network/DNS errors.

The important point: the closeout phase should not have launched any nested `codex exec` at all. It was already the runner-launched phase process.

## Likely Cause

The closeout prompt/reference currently says the phase should:

```text
Update runner telemetry receipt.
```

That wording is ambiguous. In this run, the closeout agent interpreted it as permission or obligation to validate/launch a nested runner/Codex process. The correct closeout behavior should be file-based:

- read compact execute evidence;
- update the program ledger;
- write a closeout receipt;
- return schema-valid JSON to the outer runner.

It should not launch `codex exec`, spawn a local runner, probe Codex availability, or test nested sampling.

## Desired Behavior

Architecture-program runner phases should be single-level. The outer Python runner launches one fresh `codex exec` per phase. The phase agent should never launch another `codex exec` or re-enter the local runner unless a future explicit feature designs that behavior.

For closeout specifically:

- It should reconcile compact execution evidence into the program ledger.
- It should write or update a compact closeout/goal-run telemetry receipt as a file artifact.
- It should not test Codex runtime availability.
- It should not use nested `CODEX_HOME` workarounds.
- It should not require network access.
- It should be able to run under the base `workspace-write` sandbox.

## Agent Brief

**Category:** bug
**Summary:** Prevent architecture-program closeout phases from launching nested Codex processes.

**Current behavior:**
When a local architecture-program runner reaches closeout, the closeout phase can interpret telemetry guidance as a need to probe or launch nested Codex. In a managed sandbox, that nested Codex call can fail because the default Codex state database is read-only; using a temporary Codex home then reaches network-restricted sampling failures. The result is a failed closeout even though execute completed, commits exist, validation passed, and the worktree is clean.

**Desired behavior:**
Closeout should be a file-based reconciliation phase. It should consume existing runner state, execute receipt data, validation/review summaries, and compact evidence paths. It should update the program ledger and write the closeout phase receipt directly. It must not launch `codex exec`, run the local runner recursively, create a temporary Codex home, or probe nested model availability.

**Key interfaces:**

- Local architecture program runner phase prompt construction should explicitly forbid nested `codex exec` and local-runner launches from inside every phase.
- Local runner reference documentation should clarify that closeout telemetry means writing compact file evidence, not probing runtime or launching another agent.
- Closeout phase guidance should say that the agent is already running inside the phase process and should return schema-valid phase JSON directly.
- Tests for prompt/reference behavior should cover the no-nested-runner constraint for closeout and, ideally, all phases.
